## 🆘 故障排查

### 常见问题

**Q: 系统无法启动？**
A: 检查日志文件，确保数据库目录存在，并且端口 8080 未被占用。

**Q: API 无响应？**
A: 验证 API 服务状态，检查网络连接，查看系统日志。

**Q: 数据丢失？**
A: 启用自动备份，定期导出关键数据，使用版本控制。

**Q: 性能下降？**
A: 检查 CPU/内存使用，清理过期数据，优化数据库查询。

更多解决方案，请查看 [QUICK_START.md](./docs/QUICK_START.md) 的故障排查部分。
